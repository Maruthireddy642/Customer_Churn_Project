# 📊 Customer Churn Prediction

An end-to-end machine learning project that predicts whether a telecom customer is likely to **churn** (leave the service) based on their demographic details, account information, and subscribed services. The trained model is served through a **FastAPI** REST API and is fully containerized with **Docker**.

---

## 📌 Overview

Customer churn — when a customer stops doing business with a company — is one of the most expensive problems for subscription-based businesses, since acquiring a new customer typically costs far more than retaining an existing one.

This project builds a classification model that flags customers at risk of churning, using their tenure, contract type, billing information, and the services they've subscribed to. The result is exposed as a simple, production-style API that returns a churn prediction along with a probability score.

---

## 🎯 Problem Statement

Given a customer's profile — demographics, account tenure, contract type, billing details, and subscribed services — predict whether that customer will churn (`Yes`) or stay (`No`), so the business can proactively target at-risk customers with retention offers.

## 📚 Dataset

The dataset (`churn_dataset.csv`) contains customer-level records with the following fields:

| Category | Features |
|---|---|
| **Identifier** | `customerID` |
| **Demographics** | `gender`, `SeniorCitizen`, `Partner`, `Dependents` |
| **Account info** | `tenure`, `Contract`, `PaperlessBilling`, `PaymentMethod`, `MonthlyCharges`, `TotalCharges` |
| **Services** | `PhoneService`, `MultipleLines`, `InternetService`, `OnlineSecurity`, `OnlineBackup`, `DeviceProtection`, `TechSupport`, `StreamingTV`, `StreamingMovies` |
| **Target** | `Churn` (`Yes` / `No`) |

---

## ⚙️ Machine Learning Pipeline (`train.py`)

1. **Data Cleaning**
   - Drops the non-predictive `customerID` column
   - Converts `TotalCharges` to numeric and removes rows with missing values

2. **Feature Encoding**
   - One-hot encodes all categorical columns via `pd.get_dummies(drop_first=True)`

3. **Train/Test Split**
   - 80/20 split, stratified on the target class

4. **Feature Scaling**
   - Standardizes features with `StandardScaler`

5. **Class Balancing**
   - Applies **SMOTE** (Synthetic Minority Over-sampling Technique) to the training set to correct class imbalance between churned and retained customers

6. **Model Training**
   - Trains an **XGBoost Classifier** (`n_estimators=50`, `max_depth=4`, `learning_rate=0.1`)

7. **Evaluation**
   - Reports accuracy on the held-out test set

8. **Artifact Export**
   - Saves the trained model, scaler, and the exact feature ordering (`churn_model.pkl`, `scaler.pkl`, `feature_names.pkl`) so the API can reproduce identical preprocessing at inference time

## 🌐 API (`app.py`)

The model is served with **FastAPI**. On startup, it loads the model, scaler, and feature list, and exposes the following endpoints:

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Health/welcome message confirming the API is running |
| `GET` | `/health` | Reports whether the model loaded successfully |
| `POST` | `/predict` | Accepts a customer's details and returns a churn prediction |

### Example request

```bash
curl -X POST "http://localhost:8000/predict" \
  -H "Content-Type: application/json" \
  -d '{
    "gender": "Female",
    "SeniorCitizen": 0,
    "Partner": "Yes",
    "Dependents": "No",
    "tenure": 12,
    "PhoneService": "Yes",
    "MultipleLines": "No",
    "InternetService": "DSL",
    "OnlineSecurity": "Yes",
    "OnlineBackup": "No",
    "DeviceProtection": "No",
    "TechSupport": "No",
    "StreamingTV": "No",
    "StreamingMovies": "No",
    "Contract": "Month-to-month",
    "PaperlessBilling": "Yes",
    "PaymentMethod": "Electronic check",
    "MonthlyCharges": 55.5,
    "TotalCharges": 660.0
  }'
```

### Example response

```json
{
  "prediction": "Churn",
  "churn_probability": 0.812
}
```

At inference time, the incoming request is one-hot encoded, reindexed against the training-time feature list (missing columns filled with 0), scaled with the saved `StandardScaler`, and passed to the XGBoost model — mirroring the exact preprocessing used in `train.py`.

Interactive API documentation (Swagger UI) is automatically available at `/docs` once the server is running.

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/Maruthireddy642/Customer_Churn_Project.git
cd Customer_Churn_Project
```

### 2. Create a virtual environment

**Linux/Mac**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Windows**
```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Train the model (optional — pre-trained artifacts are already included)

```bash
python train.py
```

This regenerates `churn_model.pkl`, `scaler.pkl`, and `feature_names.pkl`.

### 5. Run the API

```bash
uvicorn app:app --reload
```

The API will be available at `http://localhost:8000`, with interactive docs at `http://localhost:8000/docs`.

---

## 🐳 Running with Docker

**Build the image**
```bash
docker build -t customer-churn-api .
```

**Run the container**
```bash
docker run -p 8000:8000 customer-churn-api
```

The API will be accessible at `http://localhost:8000`.
